# タグ・友だち情報機能（Lステップ風）— 要件定義

> 作成: 2026-06-06 / 対象: FutaMindMap
> 目的: **Lステップの構成を設計・提案する図に「タグ」「友だち情報」を載せる**。
> 実際の友だち管理DBではなく、提案図としての視覚表現が主眼。

---

## 1. 全体像

- 既存の **ノード** に「タグ」「友だち情報」をひも付けできる（ノート専用ではなく**どのノードでも**可）。
- マップ単位で **2つのマスタ** を持つ：
  1. **タグマスタ**（名前＋色＋グループ）
  2. **友だち情報マスタ**（項目名のみ）
- マスタは1か所で共通管理。あるノードで作ったタグ/項目は、**他のノードからも選べる**（表記ゆれ防止）。
- 各ノードへの付与状態は、ノード選択時にツールバーの **「🏷️タグ/情報」ボタン**から開くポップアップで編集。
- ノードには付与状況を示す **2種のバッジ**（🏷️タグ数 / 📝情報数）を表示。
- エクスポート（SVG/PNG）には **バッジまで反映**（ポップアップ中身は出ない）。

---

## 2. データモデル

### 2-1. マップに追加するマスタ（`MindMap` に新フィールド）

```ts
// タグマスタ
interface TagGroup {
  id: string;
  name: string;          // グループ名（例: "ステータス", "流入経路"）
}
interface TagDef {
  id: string;
  name: string;          // タグ名（例: "見込み客"）
  color: string;         // バッジ色 (#RRGGBB)
  groupId: string | null;// 所属グループ（未分類は null）
}

// 友だち情報マスタ（項目名のみ・型なし）
interface FriendFieldDef {
  id: string;
  name: string;          // 項目名（例: "氏名", "メール", "流入経路"）
}

interface MindMap {
  // ...既存...（注: 既存の tags?: string[] はマップ分類用で別物。機能用は tagDefs）
  tagGroups?: TagGroup[];
  tagDefs?: TagDef[];
  friendFields?: FriendFieldDef[];
}
```

### 2-2. ノードに追加する付与情報（`MindMapNode` に新フィールド）

```ts
interface MindMapNode {
  // ...既存...
  tagIds?: string[];        // このノードで付与するタグ（TagDef.id の配列）
  friendFieldIds?: string[];// このノードで取得/更新する友だち情報項目（FriendFieldDef.id の配列）
}
```

- ノードは **マスタの id を参照するだけ**（名前や色はマスタを正とする → マスタ変更で全ノードに反映）。
- マスタから項目を削除したら、各ノードの参照配列からも掃除する（または表示時にマスタ非存在idは無視）。

## 3. UI 仕様

### 3-1. ノードのバッジ（常時表示・別々の2バッジ）

- ノードに付与があれば、コーナー等に小さく表示：
  - **🏷️ N**（付与タグ数）
  - **📝 N**（友だち情報項目数）
- 0件の種類はバッジ非表示。
- **バッジクリックでポップアップを開ける**（ツールバー経由と同じ動作。採用）。

### 3-2. ノードのタグ/情報ポップアップ（ツールバー「🏷️タグ/情報」から開く）

選択中ノード1つに対して開く。中身は2セクション：

**タグ セクション**
- マスタのタグを **グループごとに見出し付き**で一覧表示。
- 各タグはチェックボックス的に **付与/解除**（このノードの `tagIds` に追加/削除）。付与済みは色付きバッジで強調。
- 末尾に **「＋新規タグ」**：名前・色・グループを指定してその場でマスタに追加→即このノードに付与。

**友だち情報 セクション**
- マスタの項目を一覧表示。各項目を **選択/解除**（このノードの `friendFieldIds` に追加/削除）。
- 末尾に **「＋新規項目」**：項目名を入力してマスタに追加→即このノードで選択。

### 3-3. マスタ管理画面（ヘッダー「🏷️マスタ管理」ボタン）

マップ全体のマスタを管理するモーダル。

- **タグ管理**：グループの追加/リネーム/削除、タグの追加/リネーム/色変更/グループ移動/削除。
- **友だち情報管理**：項目の追加/リネーム/削除。
- 削除時は「このタグ/項目は N 個のノードで使われています」と警告し、削除すると各ノードからも外れる。

### 3-4. エクスポート

- SVG/PNG 書き出し時、ノードの **🏷️N / 📝N バッジは描画に含める**。
- ポップアップやマスタ管理の中身は含めない（画像なので展開不可）。

---

## 4. 保存・互換

- マスタ（`tagGroups/tags/friendFields`）とノードの付与（`tagIds/friendFieldIds`）は、
  既存の Firestore 保存フロー（`saveNodes` / マップドキュメント更新）に乗せる。
- 旧データ（これらのフィールドが無いマップ/ノード）は **未定義=空** として扱い、後方互換を保つ。
- Firestore の 1MiB 上限に配慮（タグ/項目はテキストのみで軽量なので問題なし）。

---

## 5. スコープ外（今回やらないこと）

- 実際の友だち（人）の登録・管理・人数カウント。
- 友だち情報の「型」（テキスト/日付/選択肢など）。今回は **項目名のみ**。
- タグの付与/解除の区別（**付与のみ**を扱う）。
- 条件分岐・自動化ロジックのシミュレーション。
